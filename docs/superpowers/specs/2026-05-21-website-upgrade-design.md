# Opra Website Upgrade — Design Spec
**Date:** 2026-05-21  
**Status:** Approved

---

## Goal

Upgrade the Opra website to look less AI-generated by:
1. Tightening the glass card visual treatment (Technical Precision direction)
2. Replacing all CSS transitions with Motion.js spring physics
3. Enhancing the globe with mouse parallax, arc connections, Africa-weighted dots, limb glow, and `prefers-reduced-motion` support
4. Fixing broken meta assets, accessibility gaps, UX polish, and mobile layout issues

The globe background is preserved; glass is kept for legibility against it.

## Globe Improvements

### 1. Mouse Parallax
`scroll-controller.js` — track two module-level variables `mouseOffsetX = 0` and `mouseOffsetY = 0`. Add a `mousemove` listener that maps cursor position to ±0.03 of viewport fraction and stores it in those variables. In the existing `onScroll()` function, when calling `globe.setTarget(blended)`, add the mouse offset on top of the scroll preset:
```js
globe.setTarget({
  ...blended,
  offsetX: blended.offsetX + mouseOffsetX,
  offsetY: blended.offsetY + mouseOffsetY,
});
```
This ensures scroll presets and mouse offset are always combined — scroll defines the base position, mouse adds drift on top. The existing lerp smoothing in `globe.js` handles fluid interpolation.

### 2. Arc Connections on Event Pings
`globe.js` — in `spawnEvent()`, record both the origin city vec and a second target city vec (Johannesburg as origin, random destination city as target). In `drawEvents()`, after drawing the existing radial ping, draw a great-circle arc between the two points using 40 interpolated steps. Arc is drawn with `accentA(fade * 0.3)`, line width 1px, only on the front hemisphere. The existing pill label and ping animation are unchanged.

### 3. Africa-Weighted Dot Density
`globe.js` — in `makeDots()`, after generating the base fibonacci sphere, append an additional 80 dots randomly sampled within a spherical cap centred on lat -30, lon 26 (Southern Africa), radius ~18°. These extra dots use the same `{ x, y, z }` unit vector format. Total dot count stays consistent with the `cfg.dots` value by reducing the base count to `cfg.dots - 80`.

### 4. Limb Atmosphere Glow
`globe.js` — add `drawLimb(R)` called from `draw()` after `drawGlow`. Creates a radial gradient from `R * 0.85` to `R * 1.15` centred at the globe offset position. Colour stops: transparent at inner edge → `accentA(0.07)` at `R * 1.0` → transparent at outer edge. Gives the impression of an atmospheric rim without conflicting with the existing centre glow.

### 5. `prefers-reduced-motion`
`globe.js` — check `window.matchMedia('(prefers-reduced-motion: reduce)').matches` in `init()`. If true: set `cfg.speed = 0`, `cfg.particles = 0`, `cfg.showLines = false`, `eventsPerSec = 0`. Globe still renders as a static dot-sphere — present but not distracting. The existing `snap()` / `setTarget()` API is unchanged.

## Additional Improvements

### 1. Missing Meta Assets — `og-image.png` and `logo.png`
Both are referenced in `index.html` (`og:image`, `twitter:image`) and the JSON-LD schema (`logo`) but do not exist. Create both as SVG-exported PNGs using the existing brand colours:
- `og-image.png` — 1200×630px, dark background (`#07090d`), "OPRA" wordmark centred in JetBrains Mono, tagline "Operational Systems · Johannesburg" below in Geist, cyan accent line
- `logo.png` — 512×512px, same treatment as a square lockup

Both go in the project root alongside `index.html`.

### 2. Contact Form — "What are you trying to solve?" Textarea
Add an optional `<textarea>` field to `form.contact` between the email field and the services fieldset:
```html
<label>What are you trying to solve?
  <textarea name="message" rows="3" placeholder="Briefly describe the problem or system you want built"></textarea>
</label>
```
Update `api/submit.js` to pass the `message` field through alongside the existing fields. The field is optional — no `required` attribute.

### 3. Globe — Pause RAF When Tab Hidden
`globe.js` — add a `visibilitychange` listener inside the IIFE. When `document.hidden` is true, call `cancelAnimationFrame(raf)`. When the tab becomes visible again, restart the loop with `raf = requestAnimationFrame(draw)` and reset `lastT = performance.now()` to avoid a large `dt` spike on resume.

### 4. Canvas Accessibility
`index.html` — add `role="img"` and `aria-label="Animated globe showing Opra's global network"` to `<canvas id="globe">`. Screen readers will announce it as a labelled image and skip the canvas content rather than announcing an unnamed interactive element.

### 5. Form `aria-live` Region
`index.html` — add a visually hidden status div after the form:
```html
<div id="formStatus" role="status" aria-live="polite" style="position:absolute;width:1px;height:1px;overflow:hidden;clip:rect(0,0,0,0);"></div>
```
In the inline form JS, update the success and error handlers to write the message to `#formStatus` in addition to changing the button text. Screen reader users will hear the outcome announced immediately.

### 6. "Scroll to Explore" Fade-Out
`motion-controller.js` — in `init()`, add a scroll listener that fades the `.corner-tag` element to `opacity: 0` once `window.scrollY > 80`, using `Motion.animate` with a 300ms ease-out. Does not animate back on scroll-up — once the user has scrolled, the hint has served its purpose.

### 7. Process Section Mobile Grid
`index.html` CSS — change the `@media (max-width: 540px)` breakpoint that forces `.process-grid` to `grid-template-columns: 1fr` to instead keep `1fr 1fr` until 400px:
```css
@media (max-width: 540px) {
  .process-grid { grid-template-columns: 1fr 1fr; }  /* was: 1fr */
  section { padding: 80px 22px; }
}
@media (max-width: 400px) {
  .process-grid { grid-template-columns: 1fr; }
}
```

## Architecture

### Files changed

**`index.html`**
- CSS: update glass card properties (backdrop-filter, background, border-radius, border)
- CSS: remove `transition:` from all interactive elements
- CSS: remove `.fade-up`, `.fade-in`, `[data-d]` rules
- CSS: update process grid mobile breakpoint (540px → 2-col, 400px → 1-col)
- JS: remove the `IntersectionObserver` scroll-reveal block (~30 lines)
- HTML: add `role="img"` + `aria-label` to `<canvas id="globe">`
- HTML: add `#formStatus` aria-live div after the form
- HTML: add "What are you trying to solve?" textarea to `form.contact`
- HTML: add two `<script>` tags before `</body>` — Motion CDN + `motion-controller.js`
- JS: update form submit handler to write success/error to `#formStatus`

**`motion-controller.js`** (new file, ~130 lines)
```
initHovers()        — mouseenter/mouseleave springs for all 6 element types
initScrollReveals() — inView + spring stagger for all card groups and text blocks
initCornerTag()     — fades .corner-tag out after 80px scroll
init()              — DOMContentLoaded entry point, calls all three
```

**`globe.js`** — seven targeted changes:
- `init()`: add `prefers-reduced-motion` check, conditionally zero out spin/particles/events
- `init()`: add `visibilitychange` listener to pause/resume RAF loop
- `makeDots()`: reduce base count by 80, append 80 Africa-cluster dots
- `draw()`: call `drawLimb(R)` after `drawGlow(R)`
- `drawLimb()`: new function, ring-shaped radial gradient at sphere edge
- `spawnEvent()`: record origin (Johannesburg) + destination city vecs
- `drawEvents()`: draw great-circle arc between origin and destination

**`scroll-controller.js`** — one addition:
- Add `mousemove` listener that calls `globe.setTarget({ offsetX, offsetY })` with cursor mapped to ±0.03 viewport fraction

**`api/submit.js`** — one addition:
- Read and forward the `message` field from the request body alongside existing fields

**`og-image.png`** (new file, 1200×630) — branded social preview image

**`logo.png`** (new file, 512×512) — square brand lockup for JSON-LD schema

### Load order (bottom of `<body>`)
```html
<script src="globe.js"></script>
<script src="scroll-controller.js"></script>
<script src="https://cdn.jsdelivr.net/npm/motion@12/dist/motion.js"></script>
<script src="motion-controller.js"></script>
<script>/* mobile nav + form — unchanged */</script>
```

## Constraints

- Globe background must remain readable — glass is kept at `blur(3px)` minimum
- Only additive HTML changes — no existing structure removed
- `api/submit.js` updated only to pass the new `message` field through
- `prefers-reduced-motion` must be respected — Motion.js animations skip, globe freezes to static dot-sphere
- Arc connections drawn on front hemisphere only (z > -0.2) to avoid visual clutter on globe rear
- Mouse parallax offset capped at ±0.03 viewport fraction — subtle, not distracting
- Africa dot cluster uses the same unit-vector format as `makeDots()` — no changes to the render path
